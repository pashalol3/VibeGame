### Логическая проверка и исправления

1. **Роли и способности после обмена масками.** Все проверки способностей (Вор, Дворецкий, Детектив) теперь используют `currentRole` вместо исходной `role`. Это значит, что после обмена масками новый владелец роли получает её способности.
2. **Перезапуск таймеров воров.** В начале каждой ночной фазы теперь вызывается `resetThiefTimers`, который сбрасывает прогресс кражи и личные цели каждого вора.
3. **Личные цели воров.** Убрана общая переменная `room.thiefTarget`; теперь каждый вор хранит `p.thiefTarget`. Это корректно работает при 1–2 ворах и после обменов ролями.
4. **Персональная слежка.** Слежка теперь учитывается отдельно для каждого детектива (`surveillanceUsedThisRound`) — каждый детектив может объявить слежку один раз за раунд.
5. **Пенальти за ошибочное обвинение.** Реализовано правильное отслеживание: игрок теряет голос в следующем раунде, а не в текущем, и не сбрасывается сразу. Добавлено поле `penaltyRound`.
6. **Раскрытие ролей при окончании игры.** В конце игры теперь раскрываются текущие роли всех игроков, а не только исходные роли воров.
7. **Прогресс кражи.** `stealProgress` и `stolenThisRound` теперь отправляются только самому игроку (`myStealProgress`, `myStolenThisRound`), чтобы не раскрывать вора другим.
8. **Двойная фаза Итог.** Исправлена ошибка, при которой неправильное обвинение запускало вторую фазу Итог поверх первой.
9. **Закрытие обвинений по таймауту.** Если время обвинения заканчивается без большинства, обвинение автоматически закрывается.
10. **Валидация скорости движения.** На сервере добавлена проверка максимального перемещения за отведённое время.
11. **Защита от повторного старта.** Добавлена проверка `room.phase !== 'lobby'` в обработчик `startGame`.
12. **Клиентский клик по комнате.** Исправлено: при клике по комнате в ночной фазе Вор больше не одновременно выбирает игрока под курсором.
13. **Удалён неиспользуемый код.** Убраны функции `broadcast`, `checkGameOverAfterArrest` и лишний `publicState` emit.

### Зависимости

- Установлены зависимости через `npm install` (Express, Socket.IO, nodemon).
- Сервер запускается без синтаксических ошибок.

- Природные препятствия: 10 объектов (деревья/камни/инструменты) блокируют движение полностью, проверка столкновений на сервере.

### Непроходимые лабиринтные препятствия и скрытые игроки вне зоны видимости

- Препятствия переделаны с круглых «деревьев/камней» на прямоугольные стены/изгороди, похожие на лабиринт.
  - `server.js`: новая генерация `generateObstacles` создаёт 16 горизонтальных/вертикальных стен случайной длины (3,5–9 м) и толщины (1–1,6 м).
  - Стены не пересекают комнаты, не лежат в центральной зоне спавна (5 м) и не слишком близко друг к другу.
  - `public/game.js`: стены рисуются с градиентной заливкой, тенью, рамкой и внутренней «кирпичной» сеткой, что создаёт эффект лабиринта.
- Столкновения игрока со стенами теперь корректно обрабатываются на обеих сторонах:
  - `server.js`: функции `isObstacleCollision` и `resolveObstacleCollision` используют окружность-прямоугольник столкновения с итеративным выталкиванием.
  - `public/game.js`: добавлена `resolveRectCollision`; клиентское движение (`updateMovement`) теперь не позволяет проходить сквозь стены.
- Игроки вне зоны видимости больше не отображаются на карте.
  - В цикле отрисовки игроков пропускаются все персонажи, кроме своего, находящиеся дальше `visionRadius`.
  - Маркер выделенной цели тоже не рисуется, если цель вне зоны видимости.
  - `pickTargetAt` не выбирает игроков, которых сейчас не видно.
  - Из тумана войны убраны лишние «просветы» вокруг других игроков — теперь видна только собственная зона видимости и подсветка комнат.
- E2E-тест успешно проходит.
